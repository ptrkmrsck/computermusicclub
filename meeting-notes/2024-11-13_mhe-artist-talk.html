Mark Engelmann artist talk @ computermusic.club, 2024-11-13

Background

  1. Classical percussion
  2. Keyboard
    • Rhodes → synths
  3. Computer music
    • Pure Data: early 2019
    • SuperCollider: fall 2022
  4. Main projects
    1. Straya (2014–2020)
    2. rice (2016–)
    3. M. Harlan Engelmann (2021–)
  5. Current setup
    • SCD → BlackHole 16ch → Reaper (levels, limiting, reverb)
    • Only plugin outside of basic Reaper plugins is Valhalla VintageVerb

Aesthetic

  1. No livecoding, lots of advance preparation
  2. Time should flow quickly
    • Effect of drone music, but not same textures
  3. Averse to melody as focus; center timbre, texture, and harmony (given time to be heard)
  4. Want sounds presented “naturalistically” (light on compression, reverb, and EQ)
  5. Few layers, but each layer is dense (and separate)

Writing process

  1. Start from building synth, exploring every corner of possible values
  2. Decide on control scheme
    • Spectrum from fixed media to free improv
    • Computer allows you to choose exactly what you want to control or automate
  3. Build interface with controllers or GUI

Algorithmic composition

  1. Don’t like regular sequencers; started with Orca
    • Demo: “Western Electric”
  2. Adding human feel
    • Rhythmic onset delay
  3. Pattern Library (demo: pattern-demo.scd)
    • Further can link parameter values, build out complex functions, nest patterns deeply
    • Number of “performers” (demo: planks.scd)
  4. How complex does it need to be?

Harmony & tuning

  1. JI concepts: unequal spacing, history, variable # of notes per octave
    • Demo: ji-demo.scd
    • Non-octave repeating (demo: septimal whole tone stack)
    • Further JI resources:
      • Kyle Gann: “Just Intonation Explained”
      • Kyle Gann: The Arithmetic of Listening: Tuning Theory and History for the Impractical Musician
      • mannfishh (YouTube)
  2. Slow harmonic motion
    • Demo: 49th.scd
  3. Gliss (continuous pitch space)
    • Demo: “vox” in ji-demo.scd
  4. Spacing/register as harmonic effect
    • Demo: “Nath’lesse”
  5. Detuning in stereo/splay, play with speed/shape

Interacting w/ instruments, controllers, & GUI

  1. Guitar as controller
    • Imprecise, and more expressive than keyboard
    • Exploit monophonic pitch tracking with polyphonic instrument
    • Demo: 3 patches in guitar-samp.scd
  2. MIDI controllers
    • Button combos
      • Knobs that don’t activate without button
    • Granular instrument (demo: grain-rotate.scd)
      • Rhythmic onset delay
    • Albion keys (demo: rice-dnd-2024.scd)
  3. Interacting with other instruments
    • Send MIDI to hardware synth
      • Melody controlled by harmony and onsets
      • Controls over contour and speed
  4. GUI
    • Making GUI fun
    • Build the exact micro- or macro-parameters in any layout

Structure/form

  1. Scenes
    • Change instruments, processes, parameters of same instrument, etc.
  2. Linear process
    • Single parameter with large effect (demo: FM synth in 49th.scd)
    • Processes moving in opposite directions
    • Synths becoming more or less stable (demo: bells.scd)
    • Sometimes set time, often left to feel (demo: carolers.scd)
  3. Narrative structure, influenced by film/literature
    • Theme for a set
  4. Performing with other musicians
